01 — PROJECT OVERVIEW

プロジェクト概要

01
問い合わせの65%は繰り返し
サイズ・在庫・配送・返品の4パターンが全体の65%を占める。同じ質問に毎日何十件も個別返信しているのが現状。
02
営業時間外に24時間対応できない
問い合わせの約30%が夜間・休日に発生。翌営業日までの返信遅延が購買機会の損失につながっている。
03
CS担当者がルーティン対応に疲弊
定型質問への返信にリソースが集中し、クレーム・複雑案件への対応品質が低下。離職リスクも高まっている。
SOLUTION APPROACH
→
Claude APIを使った自然言語理解で「意図を読んで回答」——「Mサイズって在庫ありますか？」のように曖昧な表現でも意図を分類し、Shopify APIからリアルタイムデータを取得して回答する。
→
FAQデータベースとの連携で正確な情報を提供——サイズガイド・返品ポリシー・配送エリア等の構造化データをSupabaseで管理。AIが毎回最新の情報を参照して回答する。
→
人間エスカレーションの自動判定——顧客の怒り感情検出・3回以上の解決失敗・返金交渉の発生を検知し、Slack経由でCS担当者に即時通知。対応漏れをゼロにする。

05 — PROMPT ENGINEERING

プロンプト設計

設計方針： 顧客の感情状態を常時モニタリングし、エスカレーション判定を最優先ルールとして定義する。 Shopify APIから取得したリアルタイムデータはsystem promptのコンテキストに注入し、Claudeが誤情報を生成しないよう「必ず提供データのみを回答に使用する」ルールを明記する。
system: | あなたは[ブランド名]のカスタマーサポートAIです。 【役割】 - 顧客の問い合わせに親切・簡潔・正確に回答する - 在庫・注文・配送に関してはShopify APIの最新データを参照する - 感情的なクレームには謝罪を先に、解決策を後に提示する - 以下の場合は必ず人間CSへエスカレーション: (1) 3回以上「理解できません」と応答した場合 (2) 返金・キャンセル交渉が発生した場合 (3) 顧客が怒り・失望の感情を表現した場合 【口調】 - 丁寧だが堅すぎない。親しみやすく、でも正確に。 - 「〜でございます」は使わない。「〜です/ます」で統一。 【現在のコンテキスト】 在庫情報: {inventory_data} 注文情報: {order_data} 過去の会話: {conversation_history} エスカレーション回数: {escalation_count} --- user: | {customer_message} --- # 期待する出力形式（JSON） { "intent": "size | stock | shipping | return | other", "sentiment": "positive | neutral | negative | angry", "should_escalate": true | false, "response_text": "顧客への返答テキスト", "shopify_query": "必要な場合のみ: order_id または sku" }
