CURRENT PROBLEMS
PROBLEM 01
物件情報をコピペ→文章を考える→媒体ごとにフォーマット変更のトリプル作業が毎回発生

// SYSTEM REQUIREMENTS

システム要件定義

機能要件・非機能要件・制約条件を確定する。

FUNCTIONAL REQUIREMENTS（機能要件）
F-01 物件スプレッドシートへの行追加を検知し、自動で処理を開始する
F-02 Instagram用（140字以内＋ハッシュタグ15個）投稿文を生成する
F-03 X（Twitter）用（140字以内、ハッシュタグ3個以内）投稿文を生成する
F-04 LINE公式アカウント用（300字以内、絵文字あり）投稿文を生成する
F-05 生成文をスプレッドシートに書き込み、担当者にLINE通知を送る
F-06 承認フラグが立ったらBuffer APIへ投稿をスケジューリングする
F-07 エラー時は担当者にLINEで内容を通知し、手動対応を促す
NON-FUNCTIONAL REQUIREMENTS（非機能要件）
N-01 行追加から投稿文生成まで3分以内に完了すること
N-02 GASの実行時間制限（6分/回）を超えないよう設計すること
N-03 Claude APIのレート制限（Tier 1: 50RPM）を超えない呼び出し設計
N-04 APIキーはGASのProperties Serviceに保存（コードに直接記述禁止）
N-05 担当者がスプレッドシートの操作のみで全機能を使えること（コード不要）
N-06 全APIコールのログをシートに記録し、月次でコスト確認できること

// CLAUDE API プロンプト設計（実装コード）
あなたは不動産会社のSNSマーケターです。 以下の物件情報から、各SNS媒体に最適化された 投稿文を3種類生成してください。 【物件情報】 {{物件種別}} / {{所在地}} 価格: {{価格}}万円 間取り: {{間取り}} / {{面積}}㎡ セールスポイント: {{セールスポイント}} ターゲット: {{ターゲット}} 【出力形式】JSON形式で返すこと。 { "instagram": "140字以内＋ハッシュタグ15個", "x": "140字以内＋ハッシュタグ3個以内", "line": "300字以内・絵文字あり・行動を促すCTA" } // ルール： // - 価格は「○○万円」と明記 // - ハッシュタグは地域名必須 // - CTAは問い合わせ・内見誘導 // - 誇大広告・事実と異なる表現禁止
★ JSON形式で受け取る理由：
Claude APIのレスポンスをJSON.parseで直接スプレッドシートの各列に書き込める。 テキスト解析が不要になりエラー率が大幅に低下する。
